# Quick Task 260512-u20: Restructure README.md for v1 Milestone Summary

One-liner: README.md rewritten to lead with the v1 consumer path (load a published model from huggingface.co/project-oceania) instead of training instructions, with the snippet honestly framed as the v1 target API rather than a working example.

## What changed

Single-file, doc-only rewrite of `README.md`. Four substantive shifts:

1. **Consumer-path lead.** A new `## Load a pre-trained model` section now appears immediately after the one-sentence project lead and before the Features list. It contains the universal `AutoModelForImageClassification.from_pretrained(..., trust_remote_code=True)` snippet from the plan's `<interfaces>` block, verbatim, framed as the v1 target API every published model will conform to. The framing makes the not-yet-published status unambiguous: the snippet is described as the API "every v1 model will conform to" rather than as a working example, and a v1-status bullet directs readers to the project status page and the HF org page for current availability.

2. **Dataset list expanded from 7 to 15.** The `## Supported datasets` section now lists every importer key under `configs/dataset_import/` (`flowcamnet`, `global_uvp5net`, `isiisnet`, `jedi_oceans_cpics`, `lensless`, `medplanktonset`, `planktonset1`, `planktoscope`, `syke_ifcb_2022`, `sykezooscan2024`, `uvp6net`, `whoi-plankton`, `zoocamnet`, `zoolake`, `zooscannet`). New entries use short factual one-noun-phrase labels — no invented provenance details. Each key is formatted in backticks, exactly as a user would type it after `dataset_import=`, so the section doubles as a CLI cheat-sheet.

3. **Stale references removed.** All three references to non-existent code are gone from the document body and from the Mermaid diagram:
   - `pz_push_model` — the entire `#### 3. Push Model to Hub` subsection deleted (module does not exist; v1 publishing path will be documented elsewhere).
   - `pz_prepare_train` — removed from the Mermaid `CLI` node label (module does not exist).
   - `planktonzilla/train_node.py` — `TRAIN_NODE` Mermaid node and its `ENTRY -->|spawns| TRAIN_NODE` edge both deleted (file does not exist on disk).

4. **Empty/placeholder sections cut.** Three credibility-eroders removed wholesale:
   - The Discord badge (line 17 of the previous version).
   - The `## 📊 Performance` section (a one-paragraph stub with no numbers).
   - The `## 🏛️ Citation` section (placeholder bibtex with a stale 2024 year, no DOI, repo URL pointing at the old `deep_plankton` name).

   Each can be re-added when there is real content to put there.

## Structural changes

Training instructions are not gone — they are preserved verbatim under a clearly demarcated `## For maintainers / Train your own model` H2 that follows a horizontal-rule separator. The `### Prerequisites`, `### Installation`, `### Import a dataset`, `### Train a model`, `### Project structure` (Mermaid + tree), `### Configuration system`, `### Loss functions for imbalanced learning`, `### Experiment tracking`, and `### Development` subsections are demoted from H2 to H3 inside this maintainer block, preserving the documentation depth while signaling that consumers can skip everything below the rule. The `## Contributing` and `## Support` sections and the closing "Built with ❤️ by Inria Chile" footer are kept at H2 because they are universal-audience.

The Mermaid diagram parses as `flowchart TD` (the user manually changed it from `LR` to `TD` between planning and execution; this was preserved). All other nodes (`CFG`, `ENTRY`, `DATA_IMPORT`, `DATA`, `MODEL`, `LOSS`, `TRAIN_LOOP`, `TRACK`, `OUTPUTS`, `SCRIPTS`, `TESTS`, `ConfigFlow` subgraph) and every other edge are untouched. The clone URL on line 116 (`Inria-Chile/planktonzilla.git`, not the legacy `deep_plankton.git`) reflects a user-made change between planning and execution and was preserved.
